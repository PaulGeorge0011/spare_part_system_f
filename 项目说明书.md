# 备件管理系统 — 项目说明书

**文档版本：** 1.0  
**系统版本：** 1.0.0  

---

## 一、项目概述

### 1.1 项目名称与定位

**备件管理系统**是一套面向生产/运维场景的备件台账与领用管理信息系统，用于统一管理备件基础信息、实物与 MES 库存、领用记录及操作审计，支持多角色权限与可选的企业微信集成，便于信息管理人员统一部署与运维。

### 1.2 建设目标

- **台账统一**：货位号 + 规格型号 + MES 物料编码等基础信息集中维护，支持实物图片与多图管理。  
- **库存可追溯**：实物库存与 MES 库存可区分维护，入库/出库/领用/管理出库等操作有明确记录与时间线。  
- **领用规范**：领用需填写领用人、数量等，领用记录与库存变动、操作日志联动。  
- **权限与审计**：区分管理员与备件领用员，关键操作（增删改、领用、用户与权限变更）写入操作日志，便于审计与问题追溯。  
- **可集成**：支持账号密码登录、可选企业微信扫码登录/注册与账号绑定，便于与现有身份体系对接。

### 1.3 适用场景

- 生产或运维部门对备件进行台账管理、领用登记与库存核对。  
- 需要按时间范围查询领用记录、入库/出库记录及操作记录。  
- 需要将用户与权限、操作行为纳入统一管理与审计。

## 二、系统架构概览

### 2.1 总体架构

系统采用**前后端分离**架构：

| 层次       | 技术选型           | 说明 |
|------------|--------------------|------|
| 前端       | Vue 3 + TypeScript + Element Plus | 单页应用，负责界面与交互，通过 REST API 访问后端。 |
| 后端 API   | FastAPI (Python)   | 提供认证、备件、领用、库存、操作日志、用户管理、图片等接口。 |
| 数据库     | MySQL 8.0          | 存储备件、领用、入库/出库、用户、操作日志等业务数据。 |
| 对象存储   | MinIO              | 存储备件实物图片，支持临时图与正式图、多图。 |
| 缓存       | Redis              | 临时图片元信息、会话等（按需使用）。 |
| 可选组件   | 企业微信 API       | 扫码登录/注册、账号绑定（需配置 CorpID、Secret 等）。 |

### 2.2 部署方式

- **容器化**：提供 `docker-compose` 编排，可一键拉起 MySQL、Redis、MinIO、FastAPI、前端、Nginx 等。  
- **进程管理**：后端由 Uvicorn 运行，生产建议使用 systemd/supervisor 或 Docker 管理，并配置多 Worker 与健康检查。  
- **反向代理**：生产环境通过 Nginx 做 HTTPS、静态资源与 `/api` 反向代理，并建议对 API 文档、登录接口做访问控制或限流。

系统提供 **`/health`** 健康检查接口，便于负载均衡与监控探测。

## 三、功能模块说明

### 3.1 功能清单（面向业务）

| 模块         | 功能要点 |
|--------------|----------|
| **登录/注册** | 账号密码登录；可选企业微信扫码登录/注册；新注册用户为「待审核」，需管理员审批后使用。 |
| **备件管理** | 备件列表查询、新增/编辑/删除；货位号+规格型号唯一；支持 MES 物料编码、实物/MES 库存、多图上传（MinIO）、规格型号/MES 编码复制、列排序等。 |
| **备件领用** | 领用界面（卡片式）；选择备件、数量、领用人（必填）、备注；扣减实物库存并写入领用记录。 |
| **库存管理** | 按时间范围查询库存流水（入库、领用、管理出库等）；展示入库/出库时间、事件类型、领用人、操作人；支持导出。 |
| **记录查询** | 操作日志查询；按时间范围、操作人、模块、动作、关键字筛选；展示操作时间、摘要与详情（含 MES 编码、变更前后信息等）。 |
| **用户管理** | 管理员可见；用户列表、角色（管理员/备件领用员）、状态（待审核/已通过）；审批/拒绝、修改角色、删除用户（仅超级管理员 admin）；权限变更后强制重新登录。 |

### 3.2 角色与权限

| 角色             | 说明 |
|------------------|------|
| **超级管理员 (admin)** | 用户名为 `admin` 的账号；唯一可删除用户、将用户审批/设为「管理员」；不可删除自身。 |
| **管理员 (admin 角色)** | 可管理备件、领用、库存、操作日志、用户（审批、改角色，但不能删除用户、不能审批为管理员）。 |
| **备件领用员**    | 可进行备件领用、查看备件与库存相关数据；无用户管理、无删除备件等敏感操作。 |

权限与状态变更后，通过 JWT `token_version` 机制使该用户当前会话失效，下次操作需重新登录。

## 四、技术要点（供信息管理人员参考）

### 4.1 安全设计

- **认证**：JWT 签发与校验，密钥通过环境变量 `SECRET_KEY` 配置，生产必须使用强随机值。  
- **密码**：bcrypt 哈希存储，不明文保存。  
- **敏感配置**：数据库连接串、MinIO 密钥、企业微信 Secret 等均通过环境变量注入，不写死在代码中。  
- **生产建议**：`DEBUG=False`；生产使用 HTTPS；CORS 配置为实际前端域名；API 文档可关闭或限制访问；登录接口建议限流。

### 4.2 数据与时间

- **数据持久化**：业务数据存 MySQL；图片存 MinIO；重要配置与清单见项目内 **DEPLOY.md**（含账户与密码清单、安全与部署检查项）。  
- **时间**：后端内部使用 UTC，API 返回带时区信息（Z）；前端展示统一为北京时间（Asia/Shanghai）。

### 4.3 运维支持

- **健康检查**：`GET /health` 返回服务状态，可用于负载均衡与监控。  
- **日志**：应用日志可对接现有日志平台；生产建议 INFO/WARNING 级别及日志轮转。  
- **备份**：建议对 MySQL 做定期备份与恢复演练；MinIO 存储桶按需备份或同步。

## 五、部署与上线要点

### 5.1 环境与配置

- 生产环境变量需单独配置（可参考根目录 `.env.example`），包括但不限于：  
  `SECRET_KEY`、`DEBUG`、`DATABASE_URL`、`FRONTEND_URL`、`CORS_ORIGINS`、MinIO 相关、企业微信相关（可选）。  
- `.env` 不应提交到版本库。

### 5.2 默认账号与生产要求

- 首次启动会自动创建默认用户：**admin / admin123**（超级管理员）、**clerk / clerk123**（备件领用员）。  
- **生产必须**：上线后立即修改默认密码或禁用/删除默认领用员账号，仅保留受控账号。

### 5.3 上线前检查（摘要）

- 敏感配置已环境变量化，未使用默认密钥/密码。  
- DEBUG 关闭，SECRET_KEY 已更换，admin 默认密码已修改或已按安全策略处理。  
- CORS 仅允许实际前端域名，生产使用 HTTPS。  
- 数据库备份策略与健康检查、进程管理已落实；API 文档已关闭或已做访问限制（如不需要对外暴露）。

更完整的**账户与密码清单、安全项、部署与运维清单**见项目内 **DEPLOY.md**。
