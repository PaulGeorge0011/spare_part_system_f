# 备件管理系统 - 部署与运维准备清单

本文档从**部署后运维**和**安全**两方面列出进入部署测试阶段前需完成的准备事项。

> **部署操作步骤**：在 Linux 云服务器上从零部署的详细步骤（Docker Compose、Nginx、HTTPS、裸机要点等）请参见 **[docs/云服务器部署指南.md](docs/云服务器部署指南.md)**。

## 账户与密码清单（使用到的账户和密码）

部署与运维前请逐项核对；**生产环境必须修改默认账号密码或使用独立凭证**，且勿将本清单中填写的真实密码提交到版本库。

### 1. 应用登录账号（本系统用户表）

| 用途 | 账号 | 默认密码 | 来源/说明 | 生产建议 |
|------|------|----------|-----------|----------|
| 超级管理员 | **admin** | **admin123** | 首次启动时由 `main.py` 自动创建（若不存在） | 上线后立即修改密码；仅此账号可删除用户、审批为管理员 |
| 备件领用员 | **clerk** | **clerk123** | 同上，自动创建 | 上线后修改密码或禁用，改用自建用户 |

以上为**系统内登录用**账号，存储在数据库 `users` 表中，密码以 bcrypt 哈希存储。

### 2. 数据库

| 用途 | 账号/连接 | 密码/说明 | 来源 | 生产建议 |
|------|------------|-----------|------|----------|
| 应用连接 DB | 由 **DATABASE_URL** 指定 | 在连接串中，如 `mysql+pymysql://用户:密码@主机:3306/库名` | 环境变量 `.env` | 使用独立生产库账号与强密码，仅通过环境变量注入 |

示例：`DATABASE_URL=mysql+pymysql://spare_app:强密码@mysql:3306/spare_part_db`

### 3. MinIO（对象存储，图片）

| 用途 | 账号/Key | 密码/Secret | 来源 | 生产建议 |
|------|----------|-------------|------|----------|
| 访问 MinIO | **MINIO_ACCESS_KEY** | **MINIO_SECRET_KEY** | `config.py` 默认或环境变量 | 生产使用独立桶与新建 Access Key/Secret，不沿用默认值 |

当前代码默认：`MINIO_ACCESS_KEY=admin`，`MINIO_SECRET_KEY=06002336fwbMINIO`（仅开发用）。

### 4. Redis（若使用）

| 用途 | 连接/账号 | 密码 | 来源 | 生产建议 |
|------|-----------|------|------|----------|
| 连接 Redis | **REDIS_URL** | 可含在 URL 中，如 `redis://:密码@host:6379/0` | 环境变量 | 生产若启用 Redis，使用独立实例与密码 |

示例：`REDIS_URL=redis://:强密码@redis:6379/0`

### 5. 企业微信（可选）

| 用途 | 标识/Key | 密钥/Secret | 来源 | 生产建议 |
|------|----------|-------------|------|----------|
| 企业微信应用 | **WECHAT_CORP_ID**（企业ID） | **WECHAT_CORP_SECRET**（应用密钥） | 环境变量 | 仅在使用企业微信登录时配置；Secret 勿提交到仓库 |
| 应用 Agent | **WECHAT_AGENT_ID** | — | 环境变量 | 与上述应用一致 |
| 回调地址 | **WECHAT_REDIRECT_URI** | — | 环境变量 | 填后端回调完整 URL |

非登录账号，为应用级凭证。

### 6. 密钥（非账号，但需保密）

| 用途 | 变量名 | 默认值/说明 | 生产建议 |
|------|--------|-------------|----------|
| JWT 签发与校验 | **SECRET_KEY** | `your-secret-key-change-in-production`（仅开发） | 生产必须设为强随机字符串（如 `openssl rand -hex 32`） |

## 一、安全

### 1.1 密钥与敏感配置

| 项 | 说明 | 操作 |
|----|------|------|
| **SECRET_KEY** | JWT 签发与校验密钥，默认值仅用于开发 | **生产必须**在环境变量中设置强随机字符串（如 `openssl rand -hex 32`），且不要提交到代码库 |
| **数据库账号密码** | DATABASE_URL 中的账号密码 | 使用独立的生产库账号、强密码，仅通过环境变量注入 |
| **MinIO** | MINIO_ACCESS_KEY / MINIO_SECRET_KEY | 生产环境使用独立桶与密钥，不沿用默认值 |
| **企业微信** | WECHAT_CORP_SECRET 等 | 仅在使用企业微信登录时配置，Secret 勿提交到仓库 |

### 1.2 默认账号与密码

| 项 | 说明 | 操作 |
|----|------|------|
| **admin 账号** | 首次启动会创建默认管理员 admin / admin123 | **上线后第一时间**用 admin 登录并修改密码；或通过用户管理创建新管理员后停用/删除默认账号（若业务允许） |
| **备件领用员** | 默认 clerk / clerk123 | 同上，修改默认密码或移除默认账号 |

### 1.3 运行环境

| 项 | 说明 | 操作 |
|----|------|------|
| **DEBUG** | 当前默认 True | 生产环境设置 `DEBUG=False`，避免泄露堆栈与内部信息 |
| **API 文档** | /api/docs、/api/redoc | 生产建议通过 Nginx 限制访问（如仅内网或 IP 白名单），或设置 `docs_url=None`、`redoc_url=None` 关闭 |
| **HTTPS** | 登录、Token 传输 | 生产必须使用 HTTPS（Nginx 或负载均衡器终止 SSL），避免 Token 与密码明文传输 |

### 1.4 访问控制与网络

| 项 | 说明 | 操作 |
|----|------|------|
| **CORS** | 允许的前端来源 | 生产环境通过 `CORS_ORIGINS` 环境变量配置为实际前端域名（如 `https://spare.yourcompany.com`），不要使用 `*` |
| **登录限流** | 防暴力破解 | 建议在 Nginx 或网关层对 `/api/v1/auth/login` 做限流（如每 IP 每分钟 10 次） |
| **敏感接口** | 用户管理、删除等 | 已做角色与 admin 用户名校验，确保生产不暴露调试接口 |

### 1.5 日志与敏感信息

| 项 | 说明 | 操作 |
|----|------|------|
| **日志内容** | 避免记录密码、Token | 确认代码中未对请求体、Authorization 头做完整打印；异常日志可记录错误信息，不记录敏感参数 |
| **错误信息** | 对用户返回的 detail | 生产可统一为简短提示，详细错误仅写日志，避免泄露路径、数据库信息 |

## 二、部署与运维

### 2.1 环境变量

| 变量 | 说明 | 生产示例 |
|------|------|----------|
| **SECRET_KEY** | JWT 密钥 | 强随机字符串 |
| **DEBUG** | 是否调试模式 | False |
| **DATABASE_URL** | 数据库连接串 | mysql+pymysql://user:pass@host:3306/dbname |
| **FRONTEND_URL** | 前端地址（回调、重定向） | https://spare.yourcompany.com |
| **CORS_ORIGINS** | 允许的 CORS 来源，逗号分隔 | https://spare.yourcompany.com |
| **MINIO_***** | MinIO 地址、桶、密钥 | 按实际 MinIO 部署填写 |
| **WECHAT_***** | 企业微信（可选） | 使用企业微信登录时必填 |

详见项目根目录 **`.env.example`**，复制为 `.env` 后按环境修改，且**不要将 `.env` 提交到版本库**。

### 2.2 数据库

| 项 | 说明 | 操作 |
|----|------|------|
| **备份** | 数据持久化 | 对 MySQL 做定期备份（全量+binlog 或快照），并验证恢复流程 |
| **连接数** | 连接池与 DB 最大连接数 | 根据后端进程数与连接池配置，设置数据库 `max_connections` 留足余量 |
| **迁移** | 表结构变更 | 当前通过启动时检查列是否存在做简单迁移；若有破坏性变更，建议先备份再部署 |

### 2.3 后端进程

| 项 | 说明 | 操作 |
|----|------|------|
| **进程管理** | 避免直接跑 python 进程 | 使用 systemd、supervisor 或 Docker 管理 uvicorn/gunicorn 进程，配置重启策略 |
| **多 Worker** | 提高并发 | 生产可用 `gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app` 等，注意与 DB 连接数匹配 |
| **健康检查** | 探测服务可用 | 已有 `/health`，在 Nginx/负载均衡/容器中配置健康检查指向该接口 |

### 2.4 前端

| 项 | 说明 | 操作 |
|----|------|------|
| **构建** | 生产构建 | `npm run build`，将 `dist` 部署到 Nginx 或静态托管 |
| **API 地址** | 请求后端 | 通过 `VITE_API_BASE_URL` 指定生产 API 根路径（如 `/api/v1` 或完整域名），避免写死 localhost |
| **Base URL** | 前端路由 | 若部署在子路径（如 /spare/），需配置 `base` 与 Nginx 的 location 一致 |

### 2.5 反向代理（Nginx）

| 项 | 说明 | 操作 |
|----|------|------|
| **HTTPS** | SSL 证书 | 配置 `listen 443 ssl`、证书与私钥，并可选启用 HSTS |
| **API 代理** | /api -> 后端 | 将 `/api` 反向代理到后端服务，并传递 `X-Forwarded-For`、`X-Forwarded-Proto` 以便日志与重定向正确 |
| **前端静态** | / -> 前端 | 将根路径或指定 location 指向前端静态资源（如 dist） |
| **上传大小** | 图片上传 | 已设 `client_max_body_size 10M`，按需调整 |

### 2.6 日志与监控

| 项 | 说明 | 操作 |
|----|------|------|
| **日志级别** | 生产建议 | 使用 INFO 或 WARNING，避免 DEBUG 产生过多日志 |
| **日志轮转** | 避免占满磁盘 | 使用 logrotate 或容器日志驱动对应用与 Nginx 日志做轮转、压缩与保留策略 |
| **监控与告警** | 可用性 | 对 `/health` 做探测，失败时告警；可选对接口耗时、错误率做监控 |

### 2.7 首次上线检查

- [ ] 所有敏感配置已改为环境变量，且未使用默认密钥/密码
- [ ] DEBUG=False，SECRET_KEY 已更换
- [ ] admin 默认密码已修改或已按安全策略处理默认账号
- [ ] CORS 仅允许实际前端域名
- [ ] 生产环境使用 HTTPS
- [ ] 数据库已备份策略，健康检查与进程管理已配置
- [ ] API 文档已关闭或已做访问限制（若不需要对外暴露）

## 三、可选增强（后续迭代）

- **登录限流**：Nginx limit_req 或网关/中间件对登录接口限流  
- **操作审计**：关键操作（如删除用户、批量删除备件）已有操作日志，可定期归档或接入审计平台  
- **会话超时**：前端可配合 Token 过期时间做无操作超时跳转登录页  
- **密码策略**：注册/修改密码时强制长度、复杂度（当前仅最小长度）

完成上述安全与部署项后，即可进入部署测试阶段；测试环境建议与生产在密钥、CORS、DEBUG 等配置上保持一致策略，仅域名与数据隔离。
