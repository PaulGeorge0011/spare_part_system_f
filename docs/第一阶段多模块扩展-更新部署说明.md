# 第一阶段多模块扩展 — 工厂服务器更新部署说明（离线/无外网）

> 适用：工厂服务器不连外网。本文包含**历次累积更新**（第一阶段权限架构、通用人员角色、统一模块权限、只读权限优化、用户管理移动端、SSO 单点登录）。

---

## 一、更新思路

- **后端**：覆盖修改过的文件（含新增 `sso.py`），不重建镜像。
- **前端**：在有网环境重新 build 生成 `dist`，整目录替换服务器上的 `frontend/dist`。
- **配置**：覆盖 `.env`（含 SSO 配置）。

## 三、有网环境：准备离线包

### 1. 创建离线包（Windows PowerShell，在项目根执行）

```powershell
$R = "D:\spare_part_system"
$O = "spare_part_update_offline"

# 创建目录结构
@(
  "backend\api\app\models",
  "backend\api\app\core",
  "backend\api\app\schemas",
  "backend\api\app\api\v1",
  "backend\api\app\crud",
  "frontend"
) | ForEach-Object { New-Item -ItemType Directory -Force -Path "$O\$_" | Out-Null }

# 复制后端文件
Copy-Item "$R\backend\api\app\models\user.py"    "$O\backend\api\app\models\"
Copy-Item "$R\backend\api\app\main.py"           "$O\backend\api\app\"
Copy-Item "$R\backend\api\app\core\permissions.py" "$O\backend\api\app\core\"
Copy-Item "$R\backend\api\app\schemas\auth.py"   "$O\backend\api\app\schemas\"
Copy-Item "$R\backend\api\app\api\v1\auth.py",
          "$R\backend\api\app\api\v1\users.py",
          "$R\backend\api\app\api\v1\sso.py"    "$O\backend\api\app\api\v1\"
Copy-Item "$R\backend\api\app\crud\user.py"      "$O\backend\api\app\crud\"

# 复制 .env（含 SSO 配置）
Copy-Item "$R\.env" "$O\"

# 构建前端 dist
Push-Location "$R\frontend"
$env:VITE_BASE_PATH = "/zs2sbgl/"
npm run build
Pop-Location
Copy-Item -Recurse "$R\frontend\dist" "$O\frontend\"

# 打包
Compress-Archive -Path "$O\*" -DestinationPath "spare_part_update_offline.zip"
Write-Host "✅ 离线包已生成：spare_part_update_offline.zip"
```

## 四、工厂服务器：执行更新

项目根以 `/opt/spare_part_system` 为例，解压到 `/tmp/offline_extract`。

### 1. 备份

```bash
cd /opt/spare_part_system
sudo tar -czf /opt/backup_spare_part_$(date +%Y%m%d).tar.gz \
  backend/api/app/models/user.py \
  backend/api/app/main.py \
  backend/api/app/schemas/auth.py \
  backend/api/app/api/v1/auth.py \
  backend/api/app/api/v1/users.py \
  backend/api/app/api/v1/sso.py \
  backend/api/app/crud/user.py \
  backend/api/app/core/permissions.py \
  .env \
  frontend/dist
```

### 2. 解压

```bash
unzip spare_part_update_offline.zip -d /tmp/offline_extract
```

### 3. 覆盖后端文件

```bash
cd /opt/spare_part_system
sudo cp /tmp/offline_extract/backend/api/app/main.py             backend/api/app/
sudo cp /tmp/offline_extract/backend/api/app/models/user.py      backend/api/app/models/
sudo cp /tmp/offline_extract/backend/api/app/core/permissions.py backend/api/app/core/
sudo cp /tmp/offline_extract/backend/api/app/schemas/auth.py     backend/api/app/schemas/
sudo cp /tmp/offline_extract/backend/api/app/api/v1/auth.py      backend/api/app/api/v1/
sudo cp /tmp/offline_extract/backend/api/app/api/v1/users.py     backend/api/app/api/v1/
sudo cp /tmp/offline_extract/backend/api/app/api/v1/sso.py       backend/api/app/api/v1/
sudo cp /tmp/offline_extract/backend/api/app/crud/user.py        backend/api/app/crud/
```

### 4. 覆盖 .env（SSO 配置）

```bash
sudo cp /tmp/offline_extract/.env /opt/spare_part_system/.env
```

### 5. 替换前端 dist

```bash
sudo rm -rf /opt/spare_part_system/frontend/dist
sudo cp -r /tmp/offline_extract/frontend/dist /opt/spare_part_system/frontend/
```

### 6. 重启 FastAPI（重建容器以读取新 .env）

```bash
cd /opt/spare_part_system
sudo docker compose up -d fastapi
```

> ⚠️ 必须用 `up -d` 而非 `restart`：`restart` 不会重新读取 `.env`，`up -d` 会重建容器并加载最新环境变量。

### 7. 验证

```bash
# 确认容器内 SSO 变量已生效
sudo docker compose exec fastapi env | grep SSO

# 确认 SSO 路由已注册（日志中应有 "SSO 单点登录路由注册成功（已启用）"）
sudo docker compose logs --tail=50 fastapi | grep SSO
```

访问 `.../zs2sbgl/` 登录页，应出现「企业微信登录」和「统一认证账号登录」按钮。

### 8. 固定二维码入口链接（可选）

系统提供**固定链接**，可用于生成固定二维码，用户扫码后直接走企业微信授权并进入项目：

| 环境 | 固定链接 |
|------|----------|
| 生产（示例） | `https://yxcf-yzyy.ynzy-tobacco.com/zs2sbgl/wechat-login` |
| 通用形式 | `https://<你的域名><BASE_PATH>wechat-login` |

- 将上述链接生成**固定二维码**，用户扫码打开后会自动跳转到企业微信/统一认证授权，授权完成后进入系统。
- 若用户已登录，访问该链接会直接进入系统首页，不会再次跳转认证。
- 若未启用 SSO，访问该链接会提示「当前未启用企业微信登录」并引导至登录页。

## 六、常见问题

**Q: 为什么按钮不显示？**  
A: 前端通过 `/api/v1/auth/sso-status` 接口判断是否启用 SSO。若该接口 404，说明 `sso.py` 未部署；若 `sso_enabled=false`，说明容器未用 `up -d` 重建（`.env` 未生效）。

**Q: SSO 登录后报"首次登录已自动注册，请等待管理员审批"？**  
A: 正常流程——工号首次扫码会自动创建待审批账号，管理员在用户管理中审批即可。

**Q: SSO 登录后报"账号待审核"？**  
A: 该工号已注册但未审批，管理员去用户管理审批。

**Q: 如何给用户一个固定二维码扫码进系统？**  
A: 使用固定链接生成二维码即可：`https://<域名>/zs2sbgl/wechat-login`（如 `https://yxcf-yzyy.ynzy-tobacco.com/zs2sbgl/wechat-login`）。用户扫码后自动跳转企业微信授权，授权完成后进入项目。
