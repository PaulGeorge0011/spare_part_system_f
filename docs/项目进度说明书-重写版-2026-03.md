# 备件管理系统项目进度说明书（重写版）

## 一、项目概述

本项目面向制丝生产现场，建设“电气 + 机械”双模块一体化备件管理平台，覆盖登录认证、备件台账、库存记录、领用归还、操作审计、报表统计、用户权限等核心业务。系统已完成工厂环境部署，支持在二级目录路径下通过企业内网/新媒体代理访问。

当前部署访问路径为：

- 外网代理：`https://yxcf-yzyy.ynzy-tobacco.com/zs2sbgl/`
- 内网直连：`http://172.24.69.125:8080/zs2sbgl/`

## 二、建设目标与实施范围

### 2.1 建设目标

- 建立统一、可追溯的备件数据台账，减少手工管理成本。
- 打通“入库-领用-归还-库存变化-统计分析”的业务闭环。
- 通过模块化权限体系实现电气、机械分域管理，降低越权与误操作风险。
- 在工厂受限网络环境下，提供可离线更新、可持续迭代的部署方案。

### 2.2 实施范围

- 业务模块：电气备件、机械备件、领用归还、库存管理、操作日志、统计报表。
- 平台能力：账号体系、角色权限、企业微信登录、SSO 单点登录。
- 运行环境：Docker Compose 编排，Nginx 代理，MySQL + Redis + MinIO。

## 三、系统技术架构（结合当前代码）

### 3.1 前端架构

- 技术栈：Vue 3 + TypeScript + Vite + Pinia + Vue Router + Element Plus + ECharts。
- 核心路由：
  - 电气：`/electrical/*`
  - 机械：`/mechanical/*`
  - 登录与认证：`/login`、`/sso/callback`、`/wechat-login`
- 核心页面：
  - 备件列表：`SparePartList`、`MechanicalSparePartList`
  - 领用页面：`SparePartRequisition`、`MechanicalSparePartRequisition`
  - 库存管理：`InventoryView`
  - 报表统计：`ReportsView`
  - 用户管理：`UserManage`
  - 操作记录：`OperationLogView`

### 3.2 后端架构

- 技术栈：FastAPI + SQLAlchemy + Pydantic + Uvicorn。
- 核心接口模块：
  - `spare_parts`（电气）
  - `mechanical_spare_parts`（机械）
  - `inventory`（库存）
  - `operation_logs`（操作记录）
  - `users`（用户）
  - `auth` / `sso` / `wechat`（认证）
  - `reports`（统计）
- 核心数据模型：
  - 备件主数据（电气/机械）
  - 领用与归还日志
  - 入库与管理出库日志
  - 通用操作日志（审计）
  - 用户与权限模型（含 scopes 与扩展权限字段）

### 3.3 部署架构

- 容器服务：`fastapi`、`mysql`、`redis`、`minio`、`nginx`、`frontend-dev`、`django-admin`。
- 部署特点：
  - 支持 `frontend/dist` 静态发布（工厂无外网场景）。
  - 支持 `/zs2sbgl/` 二级目录路径。
  - 支持 WebSocket 转发用于多端数据刷新。

## 六、近期关键优化（基于当前代码状态）

### 6.1 领用页面筛选优化

- 问题：两个领用页面“存放地筛选”前端有下拉，但后端领用查询接口未完整接收/传递该参数。
- 处理：已在电气与机械领用查询链路补齐 `storage_location` 参数传递。
- 结果：点击“存放地”筛选后可正确返回过滤数据。

### 6.2 移动端消息提示优化

- 问题：登录成功提示在移动端页面停留过长，遮挡关键按钮。
- 处理：
  - 登录成功消息显示时机调整为“路由跳转前”。
  - 全局消息时长下调（移动端进一步缩短）。
- 结果：提示短停留后自动消失，按钮可及时点击，交互连续性提升。

## 七、工厂部署现状与运维建议

### 7.1 现状

- 已具备工厂代理访问能力，支持 `/zs2sbgl/` 二级目录。

## 八、安全体系建设与当前状态

### 8.1 身份认证与账号安全

- 系统支持账号密码登录、Token 登录、企业微信登录、SSO 单点登录，降低单一登录方式失效风险。
- 后端通过统一认证接口与依赖注入校验当前用户身份，未授权请求直接拒绝访问。
- 用户模型已扩展认证与权限相关字段（如 `token_version`、`permissions`、`sso_user_id`），为会话控制与后续安全策略预留能力。

### 8.2 权限隔离与最小权限控制

- 已实现电气与机械双模块权限隔离，页面菜单、路由访问、接口调用按用户权限范围控制。
- 角色能力按管理员、领用员、只读等分级，避免“全员高权限”导致的误操作与越权风险。
- 前端路由守卫与后端接口校验共同生效，防止仅通过前端隐藏造成的“可见性安全”问题。

### 8.3 数据审计与可追溯能力

- 关键业务动作（领用、归还、入库、管理出库、用户管理变更等）均有日志记录。
- 建立“业务日志 + 操作日志”双轨审计机制，支持按时间、人员、事件类型回溯。
- 接口异常链路中已引入请求标识辅助定位，提高问题追踪与责任界定效率。

### 8.4 接口与数据安全

- 后端对关键入参进行类型与业务规则校验（如数量、范围、必填字段），减少非法数据进入核心链路。
- 对库存类接口增加余额与约束判断，避免负库存、超额归还等异常业务写入。
- 生产环境可通过配置关闭调试模式，避免过度日志暴露内部实现细节。

### 8.5 部署与运维安全

- 工厂场景采用静态 `dist` + 容器化部署，减少生产环境临时构建与依赖漂移风险。
- 通过二级目录代理、Nginx 转发规则与 WebSocket 配置统一管控外部访问入口。
- 离线更新流程支持“定向文件覆盖 + 备份回滚”，降低错误更新对生产系统的影响面。

### 8.6 安全改进计划

- 增强高风险操作告警与异常行为监控（如短时高频失败登录、异常批量操作）。
- 推进敏感配置分级管理与定期轮换（如密钥、客户端密钥、访问凭证）。
- 完善安全检查清单（部署前、发布后、月度巡检），形成制度化闭环。

## 九、当前风险与改进计划

- 早期数据库表结构在新增字段后需要持续兼容校验。

### 9.2 下一阶段计划

- 继续完善异常监控与日志追踪能力，缩短故障定位时间。
- 优化移动端交互一致性（弹窗、消息、筛选操作路径）。
- 推进报表维度扩展与导出能力优化，提升管理决策支持能力。
- 形成标准化离线发版模板与版本台账，降低工厂运维操作风险。
