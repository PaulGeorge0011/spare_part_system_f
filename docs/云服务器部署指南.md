# 备件管理系统 — Linux 云服务器部署指南

本文档面向**部署测试阶段**，指导在 **Linux 云服务器**上从零完成备件管理系统的部署与访问配置。按步骤执行即可完成部署。

---

## 一、部署目标与环境要求

### 1.1 部署目标

- **运行环境**：Linux 云服务器（如阿里云 ECS、腾讯云 CVM、华为云等）
- **访问方式**：通过域名或服务器公网 IP 使用浏览器访问（建议配置 HTTPS）

### 1.2 服务器配置建议

| 场景       | CPU  | 内存 | 磁盘  | 说明           |
|------------|------|------|-------|----------------|
| 部署测试   | 2 核 | 4 GB | 40 GB | 小规模并发     |
| 正式使用   | 4 核 | 8 GB | 80 GB | 建议 SSD、定期备份 |

### 1.3 系统与软件要求

- **操作系统**：Ubuntu 22.04 LTS / CentOS 7+ / Debian 11+ 等常见 Linux 发行版  
  **若使用 CentOS 7**，建议直接参考 **[docs/CentOS7部署指南.md](CentOS7部署指南.md)**，按步骤完成系统准备、Docker 安装与部署。
- **Docker 部署（推荐）**：
  - Docker 20.10+
  - Docker Compose 2.0+
- **裸机部署（可选）**：
  - Python 3.11+
  - Node.js 20+（用于构建前端）
  - MySQL 8.0
  - Redis 7
  - MinIO 或兼容 S3 的对象存储
  - Nginx

### 1.4 网络与防火墙

- 放行端口（根据实际使用选择）：
  - **80**：HTTP（若使用 Nginx 反向代理）
  - **443**：HTTPS（若配置 SSL）
  - **8000**：仅当直接暴露后端时需放行（不推荐，建议一律走 Nginx）
- 若使用云厂商安全组，请在安全组中开放上述端口。

---

## 二、部署方式概览

本系统支持两种部署方式：

| 方式             | 适用场景           | 难度 | 说明                     |
|------------------|--------------------|------|--------------------------|
| **Docker Compose** | 推荐，部署测试/生产 | 低   | 一键启动全部服务，易维护 |
| **裸机部署**       | 无 Docker 环境     | 中高 | 需自行安装 MySQL/Redis/MinIO/Nginx 等 |

以下**先介绍 Docker Compose 部署**（推荐），再简要说明裸机部署要点。

## 三、Docker Compose 部署（推荐）

### 3.1 准备服务器与登录

1. 购买或准备一台 Linux 云服务器，记下**公网 IP**（如 `123.45.67.89`）。
2. 使用 SSH 登录（将 `root` 和 IP 替换为你的用户名与 IP）：
   ```bash
   ssh root@123.45.67.89
   ```
3. 建议先更新系统（以 Ubuntu 为例）：
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

### 3.2 安装 Docker 与 Docker Compose

**Ubuntu / Debian：**

```bash
# 安装 Docker
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

**CentOS / RHEL：**

```bash
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl start docker && sudo systemctl enable docker
docker --version && docker compose version
```

### 3.3 上传项目代码到服务器

任选其一即可：

**方式 A：使用 Git（推荐）**

在服务器上安装 Git 后执行：

```bash
cd /opt   # 或你希望放置的目录
sudo git clone <你的仓库地址> spare_part_system
cd spare_part_system
```

**方式 B：本地上传**

在本地将项目打包（排除 `node_modules`、`__pycache__`、`.env` 等），用 `scp` 或 SFTP 上传到服务器，例如：

```bash
# 在本地执行（示例路径请按实际修改）
scp -r ./spare_part_system root@123.45.67.89:/opt/
```

上传后进入项目根目录：

```bash
cd /opt/spare_part_system   # 以你实际路径为准
```

### 3.4 配置环境变量

1. 复制环境变量示例文件并编辑：
   ```bash
   cp .env.example .env
   nano .env   # 或 vi / vim
   ```

2. **必须修改的项**（生产/测试部署务必完成）：

   | 变量名 | 说明 | 示例 |
   |--------|------|------|
   | `SECRET_KEY` | JWT 密钥，必须为强随机字符串 | 使用 `openssl rand -hex 32` 生成 |
   | `DEBUG` | 生产必须关闭 | `False` |
   | `MYSQL_ROOT_PASSWORD` | MySQL root 密码 | 自设强密码 |
   | `MYSQL_PASSWORD` | 应用使用的数据库账号密码 | 自设强密码 |
   | `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD` | MinIO 控制台与 API 账号 | 自设，勿用默认 |

3. **可选但建议配置**（若已绑定域名）：
   - `CORS_ORIGINS`：前端访问地址，如 `https://spare.yourcompany.com`
   - `FRONTEND_URL`：同上，用于回调与重定向
   - `VITE_API_BASE_URL`：若前端与后端同域且通过 Nginx 代理 `/api`，可留空或设为 `/api/v1`

4. 生成 SECRET_KEY 示例：
   ```bash
   openssl rand -hex 32
   ```
   将输出填入 `.env` 的 `SECRET_KEY=`。

5. 保存退出。**注意：不要将 `.env` 提交到 Git。**

### 3.5 启动服务

在项目根目录执行：

```bash
docker compose up -d
```

首次会拉取镜像并构建，可能需要几分钟。查看服务状态：

```bash
docker compose ps
```

应看到 `mysql`、`redis`、`minio`、`fastapi`、`frontend`、`nginx`、`django-admin`、`minio-create-buckets` 等为运行状态（`celery-worker` 若存在也应为 running）。

### 3.6 验证部署

1. **检查容器健康**：
   ```bash
   docker compose ps
   docker compose logs fastapi --tail 50
   ```

2. **检查后端健康接口**（在服务器上）：
   ```bash
   curl http://127.0.0.1:8000/health
   ```
   应返回 `{"status":"ok"}` 或类似。

3. **通过浏览器访问**：
   - 若未配置域名：在浏览器打开 `http://<你的服务器公网IP>`（Nginx 默认监听 80）。
   - 若已配置 Nginx 的 `server_name` 与域名解析：打开 `http://你的域名`。

4. **登录**：使用系统默认管理员账号 `admin` / `admin123` 登录。**上线后请立即在「用户管理」中修改 admin 密码。**

### 3.7 生产环境建议：关闭 API 文档与改用多 Worker

- 生产环境建议在 FastAPI 中关闭 Swagger/ReDoc，或在 Nginx 中对 `/api/docs`、`/api/redoc` 做 IP 白名单，避免接口文档对外暴露。
- 提高并发：可将 `fastapi` 的启动命令改为多 Worker，例如在 `docker-compose.yml` 中：
  ```yaml
  command: gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
  ```
  需在 `backend/api/requirements.txt` 中加入 `gunicorn` 并在 Dockerfile 中安装。

## 四、Nginx 与 HTTPS 配置（重要）

当前 Nginx 配置位于项目内 `nginx/nginx.conf`，由 Docker Compose 挂载。若要通过域名 + HTTPS 访问，需做以下配置。

### 4.1 域名解析

在域名服务商处添加 A 记录，将你的域名（如 `spare.yourcompany.com`）指向服务器公网 IP。

### 4.2 准备 SSL 证书

- **方式 A**：使用 Let’s Encrypt 免费证书（推荐）：
  ```bash
  sudo apt install certbot   # Ubuntu/Debian
  sudo certbot certonly --standalone -d spare.yourcompany.com
  ```
  证书一般位于 `/etc/letsencrypt/live/spare.yourcompany.com/`。

- **方式 B**：使用云厂商或自购证书，将证书与私钥放到服务器某目录（如 `/opt/spare_part_system/nginx/ssl/`）。

### 4.3 修改 Nginx 配置

在 `nginx/nginx.conf` 的 `server` 块中：

1. 将 `server_name` 改为你的域名，例如：
   ```nginx
   server_name spare.yourcompany.com;
   ```

2. 增加 443 端口的 HTTPS 配置（在原有 `listen 80` 的 `server` 后追加或改为同一块内）：
   ```nginx
   listen 443 ssl http2;
   ssl_certificate     /etc/nginx/ssl/fullchain.pem;   # 证书路径，按实际修改
   ssl_certificate_key /etc/nginx/ssl/privkey.pem;    # 私钥路径
   ssl_session_timeout 1d;
   ssl_protocols TLSv1.2 TLSv1.3;
   ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
   ```

3. 若使用 Let’s Encrypt，需在 `docker-compose.yml` 中把证书目录挂载进 nginx 容器，例如：
   ```yaml
   volumes:
     - ./nginx/nginx.conf:/etc/nginx/nginx.conf
     - ./nginx/conf.d:/etc/nginx/conf.d
     - /etc/letsencrypt:/etc/nginx/ssl:ro
   ```
   并在 nginx 里用 `/etc/nginx/ssl/live/你的域名/fullchain.pem` 和 `privkey.pem`。

4. 可选：在 80 的 `server` 中做 301 跳转到 HTTPS：
   ```nginx
   return 301 https://$server_name$request_uri;
   ```

修改后重启 Nginx 容器：

```bash
docker compose restart nginx
```

### 4.4 更新应用配置

确保 `.env` 中：

- `CORS_ORIGINS=https://spare.yourcompany.com`
- `FRONTEND_URL=https://spare.yourcompany.com`

然后重启 fastapi 容器：

```bash
docker compose restart fastapi
```

## 五、裸机部署要点（无 Docker）

若不能使用 Docker，需在 Linux 上单独安装并配置各组件。

### 5.1 安装基础服务

- **MySQL 8.0**：创建数据库与用户，例如：
  ```sql
  CREATE DATABASE spare_part_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  CREATE USER 'sparepart'@'%' IDENTIFIED BY '你的密码';
  GRANT ALL ON spare_part_db.* TO 'sparepart'@'%';
  FLUSH PRIVILEGES;
  ```

- **Redis**：安装并启动，记下端口（默认 6379）和密码（若设置）。

- **MinIO**：从官网下载二进制或使用包管理器安装，创建 bucket `spareparts`，并记录 Access Key / Secret Key、Endpoint。

### 5.2 后端（FastAPI）

```bash
cd backend/api
python3 -m venv venv
source venv/bin/activate   # Linux
pip install -r requirements.txt
```

配置环境变量（同上文，但 `DATABASE_URL`、`REDIS_URL`、`MINIO_*` 等指向本机或实际地址），然后：

```bash
export $(cat .env | xargs)   # 或使用 systemd 的 EnvironmentFile
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

生产建议用 systemd 或 supervisor 管理进程，并设置多 Worker（如 gunicorn + uvicorn worker）。

### 5.3 前端（Vue）

```bash
cd frontend
npm ci
npm run build
```

将 `dist` 目录部署到 Nginx 的静态根目录。若前端与后端同域，Nginx 将 `/api` 反向代理到 `http://127.0.0.1:8000`，则前端构建时可不设 `VITE_API_BASE_URL`（默认使用相对路径 `/api/v1`）。

### 5.4 Nginx

在 Nginx 中配置：

- 根路径 `/` → 前端静态文件（如 `root /path/to/frontend/dist; try_files $uri $uri/ /index.html;`）。
- `/api/` → `proxy_pass http://127.0.0.1:8000;`，并传递 `Host`、`X-Real-IP`、`X-Forwarded-For`、`X-Forwarded-Proto`。
- 若使用 MinIO 且通过 Nginx 代理图片，可保留与当前 `docker-compose` 中类似的 `/minio/` 配置，将 `minio:9000` 改为本机 MinIO 地址。

- [ ] 所有服务容器/进程运行正常（`docker compose ps` 或 systemctl/supervisor）。
- [ ] `.env` 中已修改默认密码与 `SECRET_KEY`，且 `DEBUG=False`。
- [ ] 使用 `admin` / `admin123` 能登录，并已修改 admin 密码。
- [ ] 前端页面能打开，列表、领用、库存等接口正常。
- [ ] 图片上传与查看正常（MinIO 与 Nginx 代理无误）。

- [ ] 数据库已安排定期备份（如 mysqldump 或云盘快照）。

## 八、常见问题

**1. 前端能打开但接口 404 或 502**

- 检查 Nginx 中 `location /api/` 是否正确代理到后端（如 `http://fastapi:8000` 或本机 `http://127.0.0.1:8000`）。
- 检查后端是否监听 `0.0.0.0:8000` 且容器/进程在运行。

**2. 登录后提示 CORS 错误**

- 确认 `.env` 中 `CORS_ORIGINS` 与浏览器访问的前端地址完全一致（含协议与端口），不要使用 `*` 用于生产。

**3. 图片无法上传或无法显示**

- 确认 MinIO 容器健康，且 `minio-create-buckets` 已成功创建 `spareparts` 桶。
- 检查 Nginx 中 `/minio/` 代理与后端 `MINIO_*` 配置是否一致；上传大小是否满足（如 `client_max_body_size 10M`）。

**4. 数据库连接失败**

- 检查 `DATABASE_URL` 格式：`mysql+pymysql://用户:密码@主机:3306/库名`。
- Docker 部署时主机名应为服务名（如 `mysql`）；裸机部署时为主机或 `127.0.0.1`。
- 确认 MySQL 已放行对应用户的远程或本地连接。

**5. 如何备份数据库？**

- Docker：`docker exec spare-part-mysql mysqldump -u root -p spare_part_db > backup.sql`（密码使用 `MYSQL_ROOT_PASSWORD`）。
- 裸机：`mysqldump -u root -p spare_part_db > backup.sql`。
